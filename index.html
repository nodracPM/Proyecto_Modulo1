¿Que es Git?

Git es un sistema de control de versiones distribuido que permite gestionar y rastrear los cambios en el código fuente de un proyecto. Esto es especialmente útil en desarrollo de software, donde múltiples personas pueden trabajar en el mismo proyecto y necesitan coordinar sus contribuciones. Al usar Git, cada desarrollador puede trabajar en su propia copia del código, realizar cambios y luego fusionar sus contribuciones en el repositorio principal. Esto asegura que el proyecto mantenga un historial detallado de todas las modificaciones, facilitando el trabajo colaborativo y la organización de versiones.

El funcionamiento de Git se basa en un repositorio, que es un espacio donde se almacena el código y su historial de cambios. Este repositorio puede estar en tu máquina local o en una plataforma de alojamiento en la nube, como GitHub, GitLab o Bitbucket, que permiten la sincronización entre diferentes desarrolladores. Con Git, puedes crear ramas (branches) para desarrollar nuevas características o solucionar errores sin afectar el código principal (main branch). Una vez finalizados los cambios, puedes combinar tu rama con la rama principal, revisando y resolviendo conflictos si se producen.

Cada vez que realizas cambios en el proyecto, puedes "comitear" (commit) esos cambios. Un commit es como una "foto" del proyecto en un momento específico, con un mensaje descriptivo sobre lo que cambió. Gracias a estos commits, puedes volver a cualquier punto en el tiempo para ver cómo estaba el proyecto en esa versión específica. Esto es crucial en caso de errores o para revisar la evolución del código, y permite a los equipos llevar un seguimiento claro de cada cambio.

¿Que es GitHub?

GitHub es una plataforma de alojamiento de repositorios basada en Git que permite a los desarrolladores almacenar, gestionar y colaborar en proyectos de software. Actúa como un "repositorio en la nube" donde los usuarios pueden subir su código, rastrear cambios y colaborar con otros programadores. Al estar basada en Git, GitHub ofrece todas las funcionalidades de control de versiones, permitiendo a los usuarios trabajar en equipo, gestionar cambios y realizar un seguimiento detallado del desarrollo de un proyecto.

Además de las funciones básicas de Git, GitHub ofrece herramientas adicionales que facilitan la colaboración. Los desarrolladores pueden abrir "issues" para reportar problemas o sugerir nuevas funcionalidades, y hacer "pull requests" para proponer cambios en el código, que otros miembros del equipo pueden revisar y aprobar. GitHub también facilita la comunicación entre los colaboradores, permitiendo que se discutan los cambios directamente en la plataforma y se hagan revisiones en tiempo real.

¿Cómo inciar inicializar un repositorio local y subirlo a GitHub?

  1. Crear un nuevo repositorio en GitHub.
  2. Inicializar el repositorio local
  3. Agregar y confirmar (commit) archivos en el repositorio local.
  4. Conectar el repositorio local con el repositorio remoto en GitHub.
  5. Enviar (push) los cambios al repositorio remoto.

Comandos básicos de git & github

  • git config --global user.name "Tu Nombre": Configura tu nombre de usuario para todos los repositorios.
  • git config --global user.email "tuemail@example.com": Configura tu correo electrónico para todos los repositorios.
  • git init: Inicializa un nuevo repositorio Git en el directorio actual.
  • git status: Muestra el estado actual del repositorio, indicando archivos modificados, nuevos archivos y archivos en el área de preparación.
  • add archivo: Agrega un archivo específico al área de preparación (stage).
  • commit -m "Mensaje del commit": Guarda los cambios en el repositorio con un mensaje descriptivo.
  • git log: Muestra el historial de commits del repositorio.
  • git push: Enviá los cambios al repositorio remoto.
  • git branch: Muestra las ramas actuales en el repositorio.
  • branch nombre-rama: Crea una nueva rama.
  • checkout nombre-rama: Cambia a otra rama.
  • checkout -b nombre-rama: Crea y cambia a una nueva rama.
  • git merge nombre-rama: Fusiona la rama especificada con la rama actual.
  • remote add origin URL: Conecta el repositorio local con un repositorio remoto.
  • git push -u origin nombre-rama: Sube los cambios a la rama especificada en el repositorio remoto.
  • git pull origin nombre-rama: Trae y combina cambios desde el repositorio remoto a la rama local.
  • git clone URL: Crea una copia local del repositorio remoto.
  • git checkout -- archivo: Deshace los cambios en un archivo en el directorio de trabajo, restableciéndolo a la última versión confirmada.
